## Why
現在、`/` ルートでアクセスした場合、エージェント選択は index 1 として扱われるが、URLは `/` のままである。これにより、エージェント選択とルーティングパスが一致せず、矛盾が生じている。また、スレッド（履歴）を選択した際に、そのスレッドのエージェントIDに対応するURLパスに遷移していない。エージェントを保存する機能は正しく動作しているが、URLパスとエージェント選択の整合性を保つ必要がある。

## What Changes
- `/` ルートでアクセスした場合、エージェントを未選択状態にする（`agentId` を `null` にする）
- `/agent/{index}` パスでアクセスした場合のみ、対応するエージェントを選択する
- ルーティングパスとエージェント選択を常に同期させる
- スレッド（履歴）を選択した際に、そのスレッドのエージェントIDに対応するURLパス（`/agent/{index}`）に即座に遷移する
- 存在しないエージェントindex（例：`/agent/13`）にアクセスした場合、index 1 にフォールバックして `/agent/1` にリダイレクトする
- 既存の `/agent/[index]` ルーティングは維持する
- エージェント選択機能の動作は維持する

## Impact
- Affected specs: `ui` capability
- Affected code:
  - `src/app/page.tsx` (ルートページでのリダイレクト処理、スレッド選択時のURL同期)
  - `src/app/agent/page.tsx` (既存のリダイレクト処理を確認)
  - ルーティングロジックの整合性確保
